Rsync-imm

From Hřbety Gigantů
Jump to: navigation, search

rsync-imm is a project that revolves around the use of a stripped down, portable cygwin install, with ssh ( id_rsa, known_hosts ) and rsync.

The main component is a batch script called sync.bat that ties all of these applications together.

It is configured by editing config.bat

[edit] Scripts Authored by Flexion

[edit] config.bat

A very simple configuration file, gets called from sync.bat 

:: Configuration File
 
@echo off
 
set i=0
set failwait=10000
set remote_host=dev-imm.compcc.com
set sftp_user=pnovotnak
set _this_path=%~dp0

 
:: Unix style paths for the folders. No drive prefix, fwd slashes, trailing slash.
set remote_folder=/home/publish/
set local_folder=/Documents and Settings/Administrator/Desktop/refresh/

[edit] sync.bat

The meat of the operation. Tries to sync with the server, if that fails, it tries to ping the server until that succeeds. 

REM  |   Peter Novotnak, Flexion 2012.
@echo off
 
call "%~dp0config.bat"
 
echo 'Sync started!' "% class="re2">time% >> %_this_path%logs\starts.log"
:start
if "%i%"=="10" (
    echo 'SYNC FAILED!..' "% class="re2">time% >> %_this_path%failure.log"
    exit
)
 
set /A i+=1
echo %i%' attempt to sync...
 
cacls "%local_folder:/=\%*" /e /p Administrator:f
cacls "%local_folder:/=\%*" /e /p Admin:f
cacls "%local_folder:/=\%*" /e /p everyone:f
cacls "%local_folder:/=\%*" /e /p %USERNAME%:f
 
"%_this_path%rsync.exe" -rz --rsh="'%_this_path%ssh.exe' -i '%_this_path%id_rsa' -o \"StrictHostKeyChecking no\" -l '%sftp_user%'" '%remote_host%':'%remote_folder%'* '%local_folder%' 2> "%_this_path%logs\transfer.error.log"

::  |    did the command complete successfully?
if NOT "%ERRORLEVEL%"=="0" (
    echo 'sync FAILED, waiting to retry...'
	:: If not, ping until victory, then try again.
    :ping
	ping -n 1 -w 20000 %remote_host% > nul
    if NOT "%errorlevel%"=="0" (
         ping -n 5 -w 1000 localhost > nul
         goto :ping
    )
    goto :start
)
 
cacls "%local_folder:/=\%*" /e /p Administrator:f
cacls "%local_folder:/=\%*" /e /p Admin:f
cacls "%local_folder:/=\%*" /e /p everyone:f
cacls "%local_folder:/=\%*" /e /p %USERNAME%:f
 
echo success! % class="re2">time% >> %_this_path%logs\success.log
 
exit

[edit] Installer

The installer simply adds a scheduled task (the sync) and allows rsync through the firewall. 

:: Add the task to the scheduler
at 00:00 /every:m,t,w,th,f,s,su "%~dp0sync.bat"

:: Allow it through the firewall
netsh firewall set allowedprogram "%~dp0rsync.exe" rsync ENABLE

[edit] Special Notes

  • While the program is running, the files in question will not reliably accessible due to permissions bits not being set properly by rsync. ( icacls runs before and after rsync. Due to the change in the file, rsync will re-download the chunk of the file that contains the permissions bits. )
  • The destination folder on the client machine must already exist, or the script will fail.
  • The task must be scheduled as the user that will be accessing the files, or permissions will not be set properly
  • When adding the scheduled task, the command must be: 
     cmd /c start /min "imm-sync"  "C:\Documents and Settings\Administrator\Desktop\rsync\sync.bat"

[edit] Problems

  • High CPU usage when pinging the dev-imm server; ~13% on the IMM Pentium 4 boxes ( according to windows task manager. ) Seems not to cause problems for the kiosk application.
  • Windows Firewall blocks rsync.exe

[edit] Installation

  1. Extract the files from the archive
  2. Edit the configuration file to suit your needs
  3. Copy the rsync folder to the kiosk system
  4. Delete any other copies of rsync_imm.zip, or it's contents
  5. Run the setup file

Done! Everything should be all set!
